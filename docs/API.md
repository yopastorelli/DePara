# API_CONTRACT

BASE_URL=http://<HOST>:<PORT>
API_PREFIX=/api
UI_PATH=/ui

## Response conventions

- Success payloads use `success: true` where route modules implement it.
- Error payloads use `error.message` and may include `error.details`, `error.field`, `error.statusCode`.
- Path validation failures must return `400` where request input is invalid.
- Rate limit failures return `429` and include `retryAfter` plus `X-RateLimit-*` headers.
- Removed legacy update routes return `410`.

## Config and backup

| Method | Path | Contract |
|---|---|---|
| GET | `/api/config` | read product config |
| POST | `/api/config` | merge/update product config |
| GET | `/api/config/export` | export versioned operational backup |
| POST | `/api/config/import` | replace covered operational state from backup |

Exported backup includes:

- `version`
- `exportedAt`
- `sourceRuntime`
- `config`
- `scheduledOperations`
- `folders`

Import rules:

- Validate version and minimum shape before write.
- Replace covered state atomically at product-contract level.
- Preserve runtime paths outside exported data.

### Execute

| Method | Path | Required fields |
|---|---|---|
| POST | `/api/files/execute` | `action`, `sourcePath`; `targetPath` for `copy` and `move` |

Supported actions:

- `copy`
- `move`
- `delete`

Path rules:

- `sourcePath` and `targetPath` must pass `validateSafePath`.
- Delete validates read/existence before removal.
- Cross-device move fallback uses Node `copyFile` plus `unlink`, not shell commands.

### Schedule

| Method | Path | Contract |
|---|---|---|
| POST | `/api/files/schedule` | create persisted scheduled operation |
| GET | `/api/files/scheduled` | list scheduled operations |
| GET | `/api/files/schedule/:operationId` | read one scheduled operation |
| PUT | `/api/files/schedule/:operationId` | partial update |
| DELETE | `/api/files/schedule/:operationId` | delete operation |
| POST | `/api/files/schedule/:operationId/execute` | execute now |

Create requirements:

- `name`
- `frequency`
- `action`
- `sourcePath`
- `targetPath` when `action` is `copy` or `move`

State rules:

- `active=false` pauses without deleting.
- `active=true` reactivates.
- `frequency=manual` persists without arming a timer.

### Slideshow and filesystem browsing

| Method | Path | Input |
|---|---|---|
| POST | `/api/files/list-images` | JSON body: `folderPath`, optional `extensions`, `recursive` |
| POST | `/api/files/list-folders` | JSON body: `path` |
| GET | `/api/files/images/:folderPath` | `folderPath` must be percent-encoded as one URL segment |
| GET | `/api/files/image/:imagePath` | `imagePath` must be percent-encoded as one URL segment |

Preferred routes for new clients:

- Use `POST /api/files/list-images` for image lists.
- Use `POST /api/files/list-folders` for folder browsing.
- Use `GET /api/files/image/:imagePath` for serving a single image.

Compatibility rule:

- GET path params are Express 5 single-segment params. Absolute paths with `/` or `\` must be encoded with `encodeURIComponent`.
- Do not use wildcard route syntax `:param(*)`.
